Frontend in 2 Days

HTML — The Skeleton

Every webpage is built from a single .html file. HTML isn’t a programming language — it’s a markup language that tells the browser what to display.

01 The Document Structure
<!DOCTYPE html>
<html lang="en">

  <head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width,
                   initial-scale=1.0">
    <title>My Page</title>
    <link rel="stylesheet"
          href="styles.css">
  </head>

  <body>
    <h1>Hello</h1>
    <script src="app.js"></script>
  </body>

</html>
1 <!DOCTYPE html> — A declaration, not a tag. Tells the browser: “I’m HTML5, render me in Standards Mode.” Without it, browsers fall back to “Quirks Mode” (1990s rendering). Always line 1.
2 <html> — The root element. Every file has exactly one. The lang attribute helps search engines and screen readers.
3 <head> — The “brain.” Nothing here is visible on the page. It holds metadata, styles, scripts, and external links. Exactly one per document.
4 <meta charset> — Sets the character encoding. UTF-8 supports all languages (CJK, emoji, accents). Without it, non-ASCII text may appear garbled.
5 viewport meta — Tells mobile browsers “page width = device width, don’t zoom out.” Without it, phones render a tiny desktop page.
6 <title> — The text on the browser tab. Required. Also used by search engines as the clickable headline.
7 <link> — Connects external resources. rel="stylesheet" loads CSS. Also used for favicons: <link rel="icon" href="favicon.ico">. It’s a void element (no closing tag). Loads in parallel — unlike @import in CSS which loads sequentially.
8 <body> — The “body.” Everything the user sees goes here: text, images, buttons, canvas. Exactly one per document.
9 <script> — Loads JavaScript. Placed at the bottom of <body> so the browser renders HTML first, then runs JS. If placed in <head>, use the defer attribute. async downloads in parallel but executes immediately — order is not guaranteed.
The Rule of One: One <html>, one <head>, one <body>. If you accidentally write two, the browser will merge them silently — but styles and scripts may break.

JavaScript — The Soul

JS makes pages interactive. It runs in the browser (or on servers via Node.js). If you know Dart, most syntax will feel familiar — JS is also C-family, dynamically typed, and supports async/await.

01 Variables & Types
// Three ways to declare variables
let count = 0;        // mutable, block-scoped
const PI = 3.14159;  // immutable binding
var legacy = true;   // function-scoped (avoid)

// Primitive types
let str   = "hello";         // String
let num   = 42;              // Number (int+float)
let big   = 9007199254740993n; // BigInt
let flag  = true;            // Boolean
let empty = null;            // intentional nothing
let nope;                    // undefined
let id    = Symbol("id");   // unique token

// Reference types
let arr = [1, 2, 3];
let obj = { name: "Alice", age: 30 };
let Block-scoped ({}). Use this for any variable that changes. Like Dart’s var.
const The binding can’t be reassigned. But if it’s an object/array, properties inside can still change. Like Dart’s final.
var Function-scoped, gets “hoisted” to the top. Legacy — avoid in modern code. Doesn’t exist in Dart.
JS Gotcha — == vs ===
"5" == 5 is true (type coercion). "5" === 5 is false (strict). Always use === unless you specifically need coercion. Dart only has == (no coercion).

04 Array Methods (Very Common)

Transforming

const nums = [1, 2, 3, 4, 5];

// map — transform each element
// Like Dart's .map()
nums.map(n => n * 2);
// [2, 4, 6, 8, 10]

// filter — keep matching
// Like Dart's .where()
nums.filter(n => n > 3);
// [4, 5]

// reduce — accumulate to one value
// Like Dart's .fold()

// find — first match (or undefined)
// Like Dart's .firstWhere()
nums.find(n => n > 3); // 4

Mutating & Checking

const arr = ["a", "b"];

// Add/remove
arr.push("c");    // end → ["a","b","c"]
arr.pop();        // end → ["a","b"]
arr.unshift("z"); // start → ["z","a","b"]
arr.shift();      // start → ["a","b"]

// splice(index, deleteCount, ...items)
arr.splice(1, 0, "x");
// insert "x" at index 1

// Checking
arr.includes("a");  // true
arr.indexOf("b");   // 1 (-1 if not found)
arr.every(x => x); // all truthy?
arr.some(x => x);  // any truthy?

// for...of (like Dart for-in)

Promises & async/await

JS × HTML — The Bridge

When the browser parses your .html file, it builds a DOM tree in memory. The DOM is not defined anywhere in your HTML — it’s a live object model the browser constructs. JavaScript talks to the page by reading and modifying this tree.

Sources vs Elements (F12): The Sources panel shows your original static files (the “blueprint”). The Elements panel shows the live DOM tree (the “building”). If JS adds a <div> at runtime, it appears in Elements but not in Sources.
01 Finding Elements
// By ID — fastest, returns one element
const title = document.getElementById("title");

// By CSS selector — very flexible
const btn = document.querySelector(".btn");
// Returns first match

const all = document.querySelectorAll("li");
// Returns NodeList (array-like)

// By class name — returns HTMLCollection
const items = document
  .getElementsByClassName("item");

// By tag name
const paras = document
  .getElementsByTagName("p");

// Console shortcut (F12 only):
// $("selector")  = querySelector
// $$("selector") = querySelectorAll
// $0 = currently selected element
// $_ = last expression result
tip Think of document as the root of the DOM tree. Every query starts from here. querySelector accepts any CSS selector — #id, .class, div > p, [data-x].
⚠️ getElementsByClassName returns a live collection — it updates automatically when the DOM changes. querySelectorAll returns a static snapshot.

    CSS — The Clothes

    CSS (Cascading Style Sheets) controls how HTML looks. Without CSS, every page is black text on a white background. The “Cascading” means later/more-specific rules override earlier/less-specific ones.

    01 Three Ways to Apply CSS

    1. External File (Best)

    <!-- In <head> -->
<link rel="stylesheet"
      href="styles.css">

/* styles.css */
body {
  background: #f5f5f5;
  font-family: sans-serif;
}

    Cached by browser. Loads in parallel. Use this for real projects.

    2. <style> Tag

    <!-- In <head> -->
<style>
  h1 {
    color: blue;
    text-align: center;
  }
</style>

    Good for small pages. No extra HTTP request. Cannot be cached separately.

    3. Inline Style

    <p style="color: red;
          font-size: 20px;">
  Styled inline
</p>

    Highest specificity. Hard to maintain. Avoid unless dynamically set by JS.

    02 Selectors — Targeting Elements
    /* Element selector */
p { color: gray; }

/* Class selector (reusable) */
.card { border: 1px solid #ccc; }

/* ID selector (unique) */
#header { height: 60px; }

/* Attribute selector */
input[type="email"] { border-color: blue; }

/* Descendant (space) */
.nav a { text-decoration: none; }

/* Direct child (>) */
ul > li { margin: 4px 0; }

/* Pseudo-classes */
a:hover { color: red; }
li:first-child { font-weight: bold; }
input:focus { outline: 2px solid blue; }
li:nth-child(2n) { background: #f9f9f9; }

/* Pseudo-elements */
p::first-letter { font-size: 2em; }
.icon::before { content: "★ "; }

/* Combinators */
h1 + p { margin-top: 0; } /* adjacent sibling */
h1 ~ p { color: gray; }   /* general sibling */
    specificity When rules conflict, the more specific selector wins:
    inline > #id > .class > element
    1000 > 100 > 10 > 1
    Equal specificity? Last one wins (cascade).
    :hover Pseudo-classes target states: hover, focus, active, first-child, nth-child. Pseudo-elements (::before, ::after) create virtual content.
    !important color: red !important; overrides everything. Avoid it. It makes debugging a nightmare. Fix specificity order instead.
    03 The Box Model
    margin (space outside)
    border
    padding (space inside)
    content
    .box {
  /* Content dimensions */
  width: 200px;
  height: 100px;

  /* Padding — inside the border */
  padding: 16px;           /* all sides */
  padding: 10px 20px;      /* top/bottom  left/right */
  padding: 5px 10px 15px 20px; /* T R B L (clockwise) */

  /* Border */
  border: 2px solid #333;
  border-radius: 8px;      /* rounded corners */

  /* Margin — outside the border */
  margin: 20px auto;       /* auto centers horizontally */

  /* CRITICAL: by default, width does NOT
     include padding+border. This fixes it: */
  box-sizing: border-box;
}

/* Apply border-box globally (best practice) */
*, *::before, *::after {
  box-sizing: border-box;
}

    06 Flexbox — One-Dimensional Layout
    /* Parent (flex container) */
.container {
  display: flex;

  /* Main axis direction */
  flex-direction: row;       /* ← default → */
  flex-direction: column;    /* ↓ vertical ↓ */

  /* Main axis alignment */
  justify-content: center;
  justify-content: space-between;
  justify-content: space-around;

  /* Cross axis alignment */
  align-items: center;
  align-items: stretch;      /* default */

  /* Wrap */
  flex-wrap: wrap;           /* overflow to next line */

  gap: 16px;                 /* space between items */
}

/* Children (flex items) */
.item {
  flex: 1;           /* grow to fill space */
  flex: 0 0 200px;   /* fixed 200px, no grow/shrink */
  align-self: end;   /* override parent align */
  order: -1;         /* reorder visually */
}
    mental model Flexbox works on one axis at a time. justify-content controls the main axis (the direction items flow). align-items controls the cross axis (perpendicular).
    common pattern Center anything horizontally + vertically:
    display: flex;
    justify-content: center;
    align-items: center;
    flex: 1 Shorthand for flex-grow: 1; flex-shrink: 1; flex-basis: 0%. Items share available space equally. Two items with flex:1 each get 50%.
    07 CSS Grid — Two-Dimensional Layout

    Grid Container

    .grid {
  display: grid;

  /* Define columns */
  grid-template-columns: 200px 1fr 1fr;
  /* 200px fixed, then 2 equal columns */

  /* Repeat shorthand */
  grid-template-columns: repeat(3, 1fr);
  /* 3 equal columns */

  /* Auto-fit responsive */
  grid-template-columns:
    repeat(auto-fit, minmax(250px, 1fr));
  /* as many 250px+ columns as fit */

  /* Rows */
  grid-template-rows: auto 1fr auto;

  /* Gaps */
  gap: 16px;
  column-gap: 24px;
  row-gap: 12px;
}

    Grid Items

    /* Span multiple columns/rows */
.wide {
  grid-column: 1 / 3;  /* col 1 to 3 */
  grid-column: span 2; /* span 2 cols */
}

.tall {
  grid-row: 1 / 4;     /* row 1 to 4 */
}

/* Named areas (very readable) */
.layout {
  grid-template-areas:
    "header  header"
    "sidebar main"
    "footer  footer";
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.main    { grid-area: main; }
.footer  { grid-area: footer; }

/* Flexbox = 1D (row OR column)
   Grid    = 2D (rows AND columns)
   Use both — they complement! */
    08 Responsive Design

    Media Queries

    /* Base styles (mobile-first) */
.container {
  padding: 16px;
}

/* Tablet and up */
@media (min-width: 768px) {
  .container {
    padding: 32px;
    max-width: 720px;
    margin: 0 auto;
  }
}

/* Desktop */
@media (min-width: 1024px) {
  .container {
    max-width: 960px;
  }
  .sidebar {
    display: block; /* show sidebar */
  }
}

/* Print styles */
@media print {
  .no-print { display: none; }
}

    Responsive Units

    /* Viewport units */
width: 100vw;   /* 100% viewport width */
height: 100vh;  /* 100% viewport height */

/* Relative units */
font-size: 1rem;    /* root element size */
font-size: 1.2em;   /* parent's font-size */
width: 50%;         /* parent's width */

/* Modern: clamp() */
font-size: clamp(14px, 2vw, 22px);
/* min: 14px, preferred: 2vw, max: 22px
   Fluid scaling without media queries! */

/* Helpful image rule */
img {
  max-width: 100%;
  height: auto;     /* preserve ratio */
}
