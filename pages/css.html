CSS — The Skin

CSS (Cascading Style Sheets) transforms raw HTML into something beautiful. It controls how elements look — colors, layout, spacing, animations.

01Three Ways to Apply CSS

1. External (Best ✅)

<!-- In <head> -->
<link rel="stylesheet"
      href="styles.css">

/* styles.css */
body {
  background: #0f1117;
  color: #e2e8f0;
}

/* Reusable, cacheable, clean
   separation of concerns */

2. Internal (Sometimes)

<!-- In <head> -->
<style>
  h1 {
    color: royalblue;
    font-size: 2rem;
  }
</style>

/* OK for single-page sites.
   Can't be cached separately. */

3. Inline (Avoid ❌)

<p style="color: red;
   font-size: 1.5rem;">
  Text
</p>

/* Highest specificity, mixes
   style with content. Only use
   for JS-driven dynamic styles. */

02Selectors & Specificity

/* --- Basic selectors --- */
p         { }  /* type (tag) */
.card     { }  /* class */
#logo     { }  /* ID */
*         { }  /* universal */

/* --- Combinators --- */
div p     { }  /* descendant */
div > p   { }  /* direct child */
h2 + p   { }  /* adjacent sibling */
h2 ~ p   { }  /* general sibling */

/* --- Pseudo-classes --- */
a:hover   { }
li:first-child { }
input:focus    { }
li:nth-child(odd) { }

/* --- Pseudo-elements --- */
p::before { content: "→ "; }
p::after  { content: " ←"; }
::selection { background: #6c8cff; }
!
Specificity hierarchy: Inline (1000) > ID (100) > Class/pseudo-class (10) > Element (1). Higher specificity wins. Use !important sparingly.
2
Cascade order: When specificity is equal, the last-declared rule wins.

03The Box Model

margin (space outside)
border
padding (space inside)
content

box-sizing: border-box — Makes width include padding + border (not just content). Without it, a 200px box with 20px padding becomes 240px. Always set * { box-sizing: border-box; }.

04Colors & Typography

Colors

/* Named */
color: tomato;

/* Hex: #RRGGBB or #RRGGBBAA */
color: #6c8cff;
color: #6c8cff80; /* 50% opacity */

/* RGB / RGBA */
color: rgb(108, 140, 255);

/* HSL */
color: hsl(228, 100%, 71%);

/* CSS custom properties */
:root { --accent: #6c8cff; }
.btn  { color: var(--accent); }

Typography

font-family: 'Inter', sans-serif;
font-size: 1rem;     /* = 16px */
font-weight: 700;     /* bold */
line-height: 1.6;
letter-spacing: 0.02em;
text-transform: uppercase;
text-align: center;

/* Units:
   px  - absolute (avoid)
   rem - relative to root font-size
   em  - relative to parent
   vw/vh - viewport width/height */

05Display & Layout

Display Values

/* block: full width, stacks vertically */
display: block;

/* inline: content width only */
display: inline;

/* inline-block: inline flow but
   respects width/height */
display: inline-block;

/* none: removed from layout
   vs visibility: hidden → occupies space */
display: none;

Position

position: static;   /* default */

position: relative; /* offset from normal */
top: 10px;

position: absolute; /* removed from flow,
  relative to positioned ancestor */

position: fixed;    /* relative to viewport */

position: sticky;   /* hybrid: relative
  until scroll threshold, then fixed */
top: 0;

06Flexbox

.container {
  display: flex;
  flex-direction: row; /* row | column */
  justify-content: center; /* main axis */
  align-items: center;  /* cross axis */
  gap: 1rem;
  flex-wrap: wrap;
}

.item {
  flex: 1;        /* grow to fill */
  flex: 0 0 200px; /* fixed 200px */
  align-self: flex-end;
  order: 2;
}

/* Center anything: */
.centered {
  display: flex;
  justify-content: center;
  align-items: center;
}
1
Main vs Cross axis: row → main is horizontal. column → main is vertical. justify-content always works on main axis.
2
flex shorthand: flex: 1 = grow:1, shrink:1, basis:0%. flex: 0 0 200px = don't grow, don't shrink, always 200px.

07CSS Grid

.grid {
  display: grid;
  grid-template-columns:
    repeat(3, 1fr);
  gap: 1.5rem;
}

/* Responsive without media queries: */
.auto-grid {
  display: grid;
  grid-template-columns:
    repeat(auto-fit,
      minmax(280px, 1fr));
  gap: 1rem;
}

.header {
  grid-column: 1 / -1; /* full width */
}
.sidebar {
  grid-row: 2 / 4;
}
1
fr unit: "Fraction of available space." 1fr 2fr = first gets 1/3, second gets 2/3.
2
auto-fit + minmax: The holy grail of responsive grids. No media queries needed.

08Responsive Design

/* Mobile-first: base = phone,
   then add complexity for bigger screens */

.container {
  padding: 1rem;
  max-width: 1200px;
  margin: 0 auto;
}

/* Tablet: 768px and up */
@media (min-width: 768px) {
  .container { padding: 2rem; }
  .grid {
    grid-template-columns: 1fr 1fr;
  }
}

/* Desktop: 1024px and up */
@media (min-width: 1024px) {
  .grid {
    grid-template-columns:
      repeat(3, 1fr);
  }
}
1
Mobile-first: Start with simplest layout, use min-width to add complexity. Mobile gets smallest CSS.

09Transitions & Animations

Transitions

.btn {
  background: #6c8cff;
  transition: all 0.3s ease;
}
.btn:hover {
  background: #8ba3ff;
  transform: translateY(-2px);
  box-shadow: 0 4px 15px
    rgba(108,140,255,0.3);
}

/* Timing: ease, linear, ease-in,
   cubic-bezier(0.4, 0, 0.2, 1) */

Keyframe Animations

@keyframes slideIn {
  from {
    opacity: 0;
    transform: translateY(20px);
  }
  to {
    opacity: 1;
    transform: translateY(0);
  }
}

.card {
  animation: slideIn 0.4s
    ease-out forwards;
}

/* Multi-step: */
@keyframes pulse {
  0%, 100% { opacity: 1; }
  50%  { opacity: 0.5; }
}

10CSS Variables (Custom Properties)

/* Define in :root for global scope */
:root {
  --bg:     #0f1117;
  --text:   #e2e8f0;
  --accent: #6c8cff;
  --radius: 12px;
}

.card {
  background: var(--bg);
  color: var(--text);
  border-radius: var(--radius);
}

/* Dark/Light toggle via JS: */
[data-theme="light"] {
  --bg:   #ffffff;
  --text: #1a1a2e;
}
1
Why variables? Change one value, every usage updates. They cascade — override in a child and everything inside uses the new value.
2
JS access: Read: getComputedStyle(el).getPropertyValue('--accent'). Write: el.style.setProperty('--accent', '#ff0').
