HTML — The Skeleton

Every webpage is built from a single .html file. HTML isn't a programming language — it's a markup language that tells the browser what to display.

01The Document Structure

<!-- This is a complete HTML document -->
<!DOCTYPE html>
<html lang="en">

  <head>
    <meta charset="UTF-8">
    <meta name="viewport"
          content="width=device-width,
                   initial-scale=1.0">
    <title>My Page</title>
    <link rel="stylesheet"
          href="styles.css">
  </head>

  <body>
    <h1>Hello</h1>
    <script src="app.js"></script>
  </body>

</html>
1
<!DOCTYPE html> — A declaration, not a tag. Tells the browser: "I'm HTML5, render me in Standards Mode." Without it, browsers fall back to "Quirks Mode" (1990s rendering). Always line 1.
2
<html> — The root element. Every file has exactly one. The lang attribute helps search engines and screen readers.
3
<head> — The "brain." Nothing here is visible on the page. It holds metadata, styles, scripts, and external links. Exactly one per document.
4
<meta charset> — Sets the character encoding. UTF-8 supports all languages (CJK, emoji, accents). Without it, non-ASCII text may appear garbled.
5
viewport meta — Tells mobile browsers "page width = device width, don't zoom out." Without it, phones render a tiny desktop page.
6
<title> — The text on the browser tab. Required. Also used by search engines as the clickable headline.
7
<link> — Connects external resources. rel="stylesheet" loads CSS. Also used for favicons. It's a void element (no closing tag). Loads in parallel — unlike @import in CSS which loads sequentially.
8
<body> — The "body." Everything the user sees goes here: text, images, buttons, canvas. Exactly one per document.
9
<script> — Loads JavaScript. Placed at the bottom of <body> so the browser renders HTML first, then runs JS. If placed in <head>, use the defer attribute. async downloads in parallel but executes immediately — order not guaranteed.

The Rule of One: One <html>, one <head>, one <body>. If you accidentally write two, the browser will merge them silently — but styles and scripts may break.

02Common Body Elements

Text & Headings

<h1>Main Heading</h1>
<h2>Sub-heading</h2>
<!-- h1 to h6, one h1 per page -->

<p>A paragraph of text.</p>

<p>Line one<br>Line two</p>
<!-- <br> forces a line break -->

<strong>Bold</strong>
<em>Italic</em>
<span>Inline wrapper</span>

Links & Images

<!-- Hyperlink -->
<a href="https://example.com"
   target="_blank">Click me</a>

<!-- Image (void element) -->
<img src="photo.jpg"
     alt="A cat sleeping"
     width="300">

<!-- alt is read by screen readers
     for accessibility -->

Lists

<!-- Unordered (bullet points) -->
<ul>
  <li>Apples</li>
  <li>Bananas</li>
</ul>

<!-- Ordered (numbered) -->
<ol>
  <li>First step</li>
  <li>Second step</li>
</ol>

Forms & Inputs

<form action="/submit" method="POST">
  <label for="name">Name:</label>
  <input type="text" id="name"
         placeholder="Enter name">
  <input type="email"
         placeholder="you@mail.com">
  <textarea rows="4"></textarea>
  <button type="submit">Send</button>
</form>

Tables

<table>
  <thead>
    <tr>
      <th>Name</th><th>Age</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td><td>30</td>
    </tr>
  </tbody>
</table>

<div> vs Semantic Tags

<!-- div: generic container -->
<div class="card">...</div>

<!-- Semantic: meaningful to SEO
     & screen readers -->
<header>Site banner</header>
<nav>Navigation links</nav>
<main>Primary content</main>
<section>Grouped content</section>
<article>Self-contained post</article>
<footer>Copyright info</footer>

03Special Elements

<canvas> — The Blank Paper

<canvas id="sky" width="400"
        height="300">
  Fallback text if unsupported
</canvas>

<!-- Canvas displays nothing by itself.
     JS draws on it via the 2D/WebGL API.

     Libraries: PixiJS (2D),
     Three.js (3D), Chart.js -->

<ruby> — Text Annotations

<!-- NOT the Ruby language!
     Named after a small font size
     used in East Asian printing. -->

<ruby>
  漢 <rt>hàn</rt>
  語 <rt>yǔ</rt>
</ruby>

<!-- Creative use:
     <ruby>I'm fine
       <rt>I'm not</rt>
     </ruby> -->

Void (Self-closing) Elements

<!-- These have NO closing tag: -->
<br>    <!-- line break -->
<hr>    <!-- horizontal rule -->
<img>   <!-- image -->
<input> <!-- form input -->
<meta>  <!-- metadata -->
<link>  <!-- external resource -->

<!-- NEVER: <div /> — swallows all
     content after it -->

PWA Manifest

<!-- Turn site into installable app -->
<link rel="manifest"
      href="manifest.json">

// manifest.json:
{
  "name": "My App",
  "short_name": "App",
  "start_url": "/",
  "display": "standalone",
  "icons": [{ "src": "icon.png",
              "sizes": "192x192" }]
}

Attributes: Tags carry data via attributes: id (unique identifier), class (reusable styling hook), src (resource URL), href (link destination), style (inline CSS). IDs must be unique across the page; classes can repeat.
