JavaScript — The Brain

JavaScript is the only programming language browsers run natively. It turns static HTML into dynamic, interactive experiences.

01Variables & Types

// Three ways to declare variables
const PI = 3.14159;      // can't reassign
let   count = 0;          // can reassign
var   old = "avoid this";  // function-scoped (legacy)

// JavaScript's 8 types
let str    = "hello";       // string
let num    = 42;            // number (int & float)
let big    = 9007199254740993n; // bigint
let bool   = true;          // boolean
let empty  = null;          // intentional "no value"
let notSet;                 // undefined
let sym    = Symbol("id"); // unique identifier
let obj    = { key: "val" }; // object
1
const vs let: Default to const. Use let only when you need reassignment. const for objects/arrays means the reference is locked — contents can still change.
2
var is legacy: var is function-scoped (not block-scoped), gets hoisted, and can be redeclared. Modern JS uses const/let.
3
Dynamic typing: Variables have no fixed type. let x = 5; x = "hello"; is valid. typeof x returns the current type.
4
null vs undefined: Both mean "no value" but null is intentional (you set it), undefined means "never assigned" (JS set it). Always use === to compare.

02Functions

// Function declaration (hoisted)
function greet(name) {
  return `Hello, ${name}!`;
}

// Arrow function (ES6+, not hoisted)
const add = (a, b) => a + b;

// Multi-line arrow
const process = (data) => {
  const cleaned = data.trim();
  return cleaned.toUpperCase();
};

// Default parameters
function power(base, exp = 2) {
  return base ** exp;
}

// Rest parameters (...)
function sum(...nums) {
  return nums.reduce((a, b) => a + b, 0);
}
1
Hoisting: Function declarations are hoisted — you can call them before they appear. Arrow functions assigned to const are NOT hoisted.
2
Template literals: Backtick strings (`...`) support ${expression} interpolation and multi-line text.
3
Arrow vs function: Arrow functions do NOT have their own this — they inherit from the enclosing scope. Use arrows for callbacks; use function for methods.

03ES6+ Features

Destructuring

// Object destructuring
const user = { name: "Alex", age: 25 };
const { name, age } = user;

// Array destructuring
const [first, ...rest] = [1, 2, 3];

// Rename + default
const { name: n, role = "user" } = user;

Spread Operator

// Clone array
const copy = [...original];

// Merge objects
const merged = { ...defaults, ...custom };

// Pass array as arguments
Math.max(...numbers);

Optional Chaining

// Safely access nested properties
const city = user?.address?.city;
// Returns undefined if any link is null

// Nullish coalescing
const port = config.port ?? 3000;
// Uses 3000 only if port is null/undefined

Modules (import/export)

// math.js — named exports
export const PI = 3.14;
export function add(a, b) {
  return a + b;
}

// app.js — import
import { PI, add } from './math.js';

// Default export
export default class App { }

04Arrays — The Workhorse

Transform & Filter

const nums = [1, 2, 3, 4, 5];

// map: transform each item
const doubled = nums.map(n => n * 2);
// [2, 4, 6, 8, 10]

// filter: keep items that pass test
const evens = nums.filter(n => n % 2 === 0);

// find: first match
const found = nums.find(n => n > 3);

// Chain them!
const result = nums
  .filter(n => n > 2)
  .map(n => n * 10);

Reduce & Others

// reduce: collapse to one value
const total = nums.reduce(
  (acc, n) => acc + n, 0
); // 15

// some / every: boolean checks
nums.some(n => n > 4);  // true
nums.every(n => n > 0); // true

// forEach: side effects only
nums.forEach(n => console.log(n));

// includes: membership test
[1,2,3].includes(2); // true

05Async — Promises & await

// A Promise = a "future value" container

fetch('https://api.example.com/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(err => console.error(err));

// Same thing with async/await (cleaner)
async function loadData() {
  try {
    const res = await fetch('/api/data');
    const data = await res.json();
    console.log(data);
  } catch (err) {
    console.error('Failed:', err);
  }
}

// Run multiple in parallel
const [users, posts] = await Promise.all([
  fetch('/users').then(r => r.json()),
  fetch('/posts').then(r => r.json()),
]);
1
Promise states: pending → fulfilled or rejected. Once settled, a Promise never changes again.
2
async/await: Syntactic sugar over Promises. await pauses the function (not the thread!) until the Promise resolves. Always use try/catch.
3
Promise.all: Runs multiple Promises in parallel. Returns when ALL resolve. Use Promise.allSettled for all results regardless of failures.

06Classes & Objects

class Animal {
  #sound; // Private field (ES2022)

  constructor(name, sound) {
    this.name = name;
    this.#sound = sound;
  }

  speak() {
    return `${this.name} says ${this.#sound}`;
  }

  static create(name, sound) {
    return new Animal(name, sound);
  }
}

class Dog extends Animal {
  constructor(name) {
    super(name, "Woof");
  }
  fetch(item) {
    return `${this.name} fetches ${item}`;
  }
}

const rex = new Dog("Rex");
rex.speak(); // "Rex says Woof"
1
Prototype-based: Classes are syntactic sugar over JavaScript's prototype system. typeof Dog is "function".
2
Private fields (#): ES2022 added real private fields with #. They're truly private — outside access throws a SyntaxError.

07Error Handling

try {
  const data = JSON.parse(rawInput);
  processData(data);
} catch (error) {
  console.error('Parse failed:', error.message);
} finally {
  cleanup(); // Runs no matter what
}

// Custom error
class ValidationError extends Error {
  constructor(field, message) {
    super(message);
    this.field = field;
    this.name = 'ValidationError';
  }
}

throw new ValidationError(
  'email', 'Invalid email format'
);
1
Catch scope: The error parameter is only visible inside the catch block.
2
finally: Always runs — even if try succeeds or catch fires. Use it for cleanup. Even return in try won't skip finally.
